IDs and Classes

Need to be able to identify different divs and other elements of the same type

In CSS, Reference IDs with #, ie #emphasis

IDs are meant for single, specific elements, not groups of elements.

Classes are meant for groups of elements.

For example, maybe you want all Buttons to have a similar style, so you'd create a Buttons class

But if one button needs to be different, give it an ID.

Reference classes in CSS with a '.', ie .buttons or .emphasis

Try not to overdo it with IDs, easy to get messy/littered

CSS3 allows for targeting elements without identifiers (ul:first-of-type, li:nth-child(3)) but older browsers may not support it. AKA pseudo-classes.
